System.Collections.Generic.List<System.String>

An "expanded" joints list as a flat list of bone names
Remarks

The algorithm is based on this description:

An "expanded" list of joints, not just a linear array of the joints as defined in the skeleton file. In particular, any joint that has more than one child will be repeated in the list for each of its children.

Remarks

We use xml scema validation on top of the xml de-serializer, since the schema has some stricter checks than the de-serializer provides. E.g. the vector attributes are guaranteed to hold only 3 float values. This reduces the need for error checking while working with the loaded skeleton.
