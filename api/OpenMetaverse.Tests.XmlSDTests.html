DeserializeIntegers()

Test that various Integer representations are parsed correctly. These tests currently only test for values within the range of a 32 bit signed integer, even though the SD specification says the type is a 64 bit signed integer, because LLSInteger is currently implemented using int, a.k.a. Int32. Not testing Int64 range until it's understood if there was a design reason for the Int32.
